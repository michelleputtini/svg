Cos'è un file SVG

SVG sta per Scalable Vector Graphics ed è un formato per immagini vettoriali bidimensionali. A differenza dei formati raster (come PNG o JPEG), non utilizza pixel ma descrive le immagini con istruzioni testuali. Questo permette di mantenere la massima nitidezza a qualsiasi dimensione, senza perdita di qualità.

Com’è fatto: XML alla base

Un file SVG è scritto in XML, un linguaggio di markup leggibile sia da esseri umani che da software. Ogni forma è definita da un elemento (es. <circle>, <path>) con attributi come coordinate, dimensioni e colori. È possibile modificare un SVG anche con un semplice editor di testo.

Integrazione nel web

L’SVG può essere inserito direttamente nel codice HTML o richiamato come file esterno. Può essere stilizzato con CSS e reso interattivo con JavaScript. Ogni suo elemento può reagire a eventi, rendendolo ideale per UI dinamiche, animazioni e grafica responsiva.

Quando usarlo

Lo SVG è perfetto per loghi, icone, grafiche UI, illustrazioni, infografiche e animazioni web. Non è adatto a contenuti fotografici, ma è ideale per tutto ciò che deve rimanere nitido e modificabile, su qualsiasi dispositivo.

Storia del formato SVG

Origini e sviluppo

Il formato SVG nasce alla fine degli anni '90 su iniziativa del W3C (World Wide Web Consortium), con l’intento di definire uno standard aperto per la grafica vettoriale scalabile sul web. Dopo una fase di confronto tra diverse proposte, nel 2001 viene pubblicata la prima versione ufficiale: SVG 1.0.

Lenta adozione iniziale

Nonostante il potenziale, l’adozione dell’SVG è inizialmente limitata. I browser del tempo offrivano un supporto incompleto, e la grafica web era dominata da formati raster più semplici da gestire come PNG, JPEG e GIF. Per anni l'SVG rimane una tecnologia di nicchia.

La svolta con il web moderno

Con la diffusione del responsive design, dei dispositivi mobili e degli schermi ad alta densità (come i Retina display), l’interesse per formati scalabili e leggeri cresce. L’SVG risponde perfettamente a queste esigenze: immagini nitide, file leggeri, controllo via CSS e JS.

Lo standard oggi

Oggi SVG è ampiamente supportato da tutti i browser moderni e integrato in software professionali come Adobe Illustrator, Figma, Sketch e Affinity. È diventato uno standard affidabile, potente e indispensabile per la grafica web, soprattutto nei contesti responsive e interattivi.

Raster / Vettoriale

Due modi per rappresentare le immagini

Nel mondo digitale, le immagini possono essere descritte in due modi principali: raster e vettoriale. Capire la differenza tra questi approcci è fondamentale per scegliere il formato giusto.

Immagini raster: pixel per pixel

I formati raster (come JPEG, PNG, GIF) salvano l’immagine come una griglia di pixel. Ogni pixel ha un colore specifico, e l’insieme crea l’immagine visibile. Questo sistema funziona bene per fotografie e immagini ricche di dettagli, ma presenta dei limiti:

  • La qualità dipende dalla risoluzione originale
  • Ingrandendo troppo l’immagine, i pixel diventano visibili (effetto “sgranato”)
  • I file possono diventare molto pesanti, soprattutto con alte risoluzioni

Immagini vettoriali: istruzioni matematiche

Le immagini vettoriali, come quelle in formato SVG, non usano pixel. Sono costruite con forme geometriche (linee, curve, poligoni) definite da coordinate e formule. I principali vantaggi sono:

  • Scalabilità perfetta: qualità invariata a qualsiasi dimensione
  • Modificabilità: ogni elemento è indipendente e può essere modificato facilmente
  • File leggeri: ideali per icone, loghi, illustrazioni semplici e UI

Quale scegliere?

Usa raster per fotografie, texture o immagini complesse.

Vantaggi dell’SVG

Il formato SVG offre numerosi vantaggi, soprattutto nel contesto della grafica digitale moderna. Ecco perché è spesso preferito rispetto ai formati raster per determinati usi:

Qualità sempre perfetta

L’SVG resta nitido anche su display ad alta densità o in stampa, senza perdere definizione.

File leggeri

Per loghi e icone semplici, il peso è ridottissimo rispetto a formati raster.

Modificabile via codice

Essendo XML, è leggibile e versionabile anche via Git o editor di testo.

Animabile e interattivo

Ogni elemento può reagire a CSS o JavaScript: perfetto per UI dinamiche.

Accessibilità & SEO

Il testo negli SVG può essere letto dai motori di ricerca e screen reader.

Caratteristica SVG (Vettoriale) PNG / JPEG (Raster)
Qualità su zoom Sempre nitida Peggiora con l’ingrandimento
Modificabilità Alta, via codice Limitata, richiede software
Interattività Supportata con CSS/JS Non supportata
Peso del file Molto basso (icone, loghi) Più pesante a seconda della risoluzione
Uso ideale Loghi, UI, icone, grafici Foto, texture, immagini realistiche
